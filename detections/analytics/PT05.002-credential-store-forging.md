# Detection: Credential Store Forging — Anomalous Credential Cache Write Patterns

## Objective

Detect forged credential material being planted in OS credential stores by establishing baselines for credential cache write patterns and alerting on writes that are inconsistent with normal authentication workflows.

---

## Analytic Logic

### Analytic 1 — Credential Manager Write Outside Authentication Event

Track writes to Windows Credential Manager that are not preceded by a successful authentication event from the same process within a 60-second window:

```
Window: 60 seconds
Sequence:
  1. CredWrite() API call from process X
  2. No preceding authentication event (Event ID 4624) associated with process X
Alert: Credential write without preceding auth event
Exclude: Known password managers (1Password, Bitwarden, KeePass)
```

### Analytic 2 — LSA Cache Write Volume Anomaly

Baseline the volume of LSA credential cache writes per day. Alert when writes exceed 2x the baseline in a single day — indicative of credential store stuffing.

```
Baseline period: 14 days
Aggregation: count(LSA cache writes) per day
Alert: observed_count > baseline_mean * 2
```

### Analytic 3 — Kerberos Ticket with Anomalous Lifetime

Alert on Kerberos tickets with lifetimes, encryption types, or PAC contents inconsistent with your DC configuration — indicative of forged ticket injection.

```
Data source: Windows Security Event Log (Event ID 4769)
Alert conditions:
  - ticket_encryption_type not in [0x12, 0x11]  # AES256/AES128 only
  - ticket_lifetime_hours > DC_policy_max_lifetime
  - ticket_renewable_lifetime_days > DC_policy_max_renewable
```

---

## Tuning Notes

- Analytic 1 requires API-level monitoring (ETW or EDR) — standard Windows event logs do not capture CredWrite calls
- Analytic 3 requires Kerberos audit logging enabled (Advanced Audit Policy: Account Logon > Kerberos Service Ticket Operations)
- Forged golden tickets often have very long lifetimes (10 years) — the lifetime anomaly check is high-fidelity

---

## Validation

1. Use Mimikatz to inject a forged silver ticket with a non-standard encryption type
2. Confirm Analytic 3 fires on the anomalous encryption type
3. Write a test credential to Credential Manager outside an authentication context and confirm Analytic 1 fires
