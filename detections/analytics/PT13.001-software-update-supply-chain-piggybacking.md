## Objective

Detect malicious software updates delivered via a compromised update channel by establishing behavioural baselines for post-update application behaviour and alerting on applications that exhibit anomalous behaviour immediately after an update event.

---

## Analytic Logic

### Analytic 1 — Post-Update Network Connection Anomaly

Baseline the network connections established by each application in the 5 minutes following an update event. Alert when a post-update network connection is made to a destination not in the application's historical connection profile.

```
Baseline period: 90 days of post-update network behaviour per application
Window: 5 minutes post-update-event
Alert: new_destination not in historical_connection_profile
        AND application_name in low_popularity_app_list
```

### Analytic 2 — Mass Simultaneous Update Installation Across Endpoint Fleet

Alert when the same application update installs across more than 20% of the endpoint fleet within a 2-hour window — especially for low-popularity applications with small install bases.

```
Window: 2 hours
Aggregation: count(update_installed) GROUP BY application_name, version
Alert: count > total_install_base * 0.20
       AND application_popularity_rank > 100  -- low popularity apps
```

### Analytic 3 — Update Binary Signing Certificate Change

Alert when an application update is signed by a different code signing certificate than previous versions — indicating potential signing key compromise.

```
Data source: Software inventory / EDR
Alert: update_signing_cert_thumbprint != previous_update_signing_cert_thumbprint
       AND application_name = same application
```

---

## Tuning Notes

- Analytic 1 is most effective for internal/LOB applications with stable, well-defined network behaviour
- Analytic 2 threshold (20%) should be calibrated — large enterprise deployments with many endpoints may have high simultaneous update rates for popular software
- Analytic 3 is high-fidelity — legitimate certificate changes are infrequent and always pre-announced by vendors

---

## Validation

1. Deploy a test application update to a subset of endpoints
2. Have the update establish a new outbound network connection to a test destination
3. Confirm Analytic 1 fires for the new post-update destination
4. Re-sign the update binary with a different test certificate and confirm Analytic 3 fires
