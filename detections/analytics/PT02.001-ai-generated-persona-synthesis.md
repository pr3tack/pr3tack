## Objective

Score inbound contractor, vendor, and external collaborator identities against a set of statistical anomaly indicators consistent with AI-generated synthetic personas, enabling prioritised manual review before access is granted.

---

## Analytic Logic

This is a **scoring model** rather than a binary rule. Each indicator contributes a score; identities exceeding a threshold are flagged for manual review.

### Scoring Model

| Indicator | Score | Rationale |
|---|---|---|
| LinkedIn account created < 6 months ago | +20 | Synthetic personas are often freshly created |
| Profile photo returns no reverse image search results | +25 | AI-generated headshots have no indexed duplicates |
| Profile photo detected as AI-generated by classifier | +30 | Direct indicator of synthetic identity |
| Connection count < 50 | +10 | Low connection count for claimed experience level |
| All connections created within same 30-day window | +20 | Botnet-seeded connection farming pattern |
| Employment history contains non-verifiable companies | +15 per company | Ghost employers used to bulk out fake history |
| Cross-platform profile inconsistencies (name/photo/dates) | +25 | Synthetic identities often have subtle cross-platform discrepancies |
| No public work product (commits, papers, posts) despite claimed seniority | +15 | Senior professionals typically have verifiable output |
| Email domain registered < 90 days ago | +20 | Fresh domain registration consistent with adversary infrastructure |
| Writing style anomaly score (LLM-generated text detector) | +15 | AI-written communications lack stylistic variation |

**Threshold:**
- Score ≥ 40: Flag for Security review before any system access
- Score ≥ 70: Flag for CISO/Legal review; apply zero-trust onboarding protocol
- Score ≥ 90: Escalate as suspected synthetic persona; do not grant access pending investigation

### Implementation (Pseudo-code)

```python
def score_identity(profile: ExternalContactProfile) -> int:
    score = 0

    if profile.linkedin_account_age_days < 180:
        score += 20
    if not profile.photo_reverse_image_match:
        score += 25
    if profile.photo_ai_generated_confidence > 0.8:
        score += 30
    if profile.linkedin_connection_count < 50:
        score += 10
    if profile.connections_created_in_30_day_window > 0.7:
        score += 20
    for employer in profile.employment_history:
        if not employer.is_verifiable:
            score += 15
    if profile.cross_platform_inconsistencies:
        score += 25
    if profile.claimed_seniority == "senior" and not profile.public_work_product:
        score += 15
    if profile.email_domain_age_days < 90:
        score += 20
    if profile.writing_style_ai_score > 0.7:
        score += 15

    return score
```

## Tuning Notes

- **Tooling integration:** Feed this scoring model into your HR onboarding workflow, vendor risk management platform, or identity governance tool
- **AI photo detection:** Integrate [Hive Moderation API](https://thehive.ai/), [AI or Not](https://www.aiornot.com/), or [Illuminarty](https://illuminarty.ai/) for headshot classification
- **Cross-platform checks:** Use OSINT tooling (Maltego, SpiderFoot) to automate cross-platform consistency checks
- **Threshold calibration:** Calibrate thresholds against your historical onboarding data to set a false-positive rate acceptable to HR. Start conservative (≥70 for Security review) and lower over time as confidence grows
- **DPRK applicant context:** For organisations in technology, finance, or defence, raise baseline sensitivity — North Korean state-sponsored IT worker campaigns specifically use synthetic or stolen identities to gain employment

---

## Validation

1. Create a test profile using an AI-generated headshot and a freshly created LinkedIn account
2. Submit the test profile through your onboarding workflow
3. Confirm the scoring model returns a score ≥ 40 and the review flag is triggered
4. Validate that a legitimate new hire profile scores below the threshold
