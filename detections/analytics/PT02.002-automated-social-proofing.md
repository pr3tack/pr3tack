## Objective

Identify coordinated inauthentic behaviour (CIB) patterns in the social graph of an inbound external contact — detecting adversary-manufactured legitimacy through botnet-driven endorsements and engagement farming before a social engineering operation succeeds.

---

## Analytic Logic

### CIB Pattern Indicators

| Indicator | Weight | Rationale |
|---|---|---|
| ≥ 10 profile connections created on the same day | High | Botnet seeding creates burst connection patterns |
| > 60% of endorsers have accounts created within 90 days | High | Bot network accounts are typically freshly created |
| GitHub stars received from accounts with zero prior activity | High | Star farming by disposable bot accounts |
| LinkedIn endorsements from profiles with no mutual industry context | Medium | Endorsers outside the professional domain are suspect |
| Engagement ratio > 50:1 (endorsements:posts) | Medium | High endorsements relative to content output suggests farming |
| All positive reviews/endorsements added within a 72-hour window | High | Coordinated burst indicative of bot orchestration |
| Endorsing accounts share similar account creation dates (±7 days) | High | Bot network registration clustering |
| GitHub repository stars-to-commits ratio > 100:1 | Medium | Star farming on low-activity repositories |

### Social Graph Analysis (Pseudo-code)

```python
def detect_cib(contact: ExternalContactProfile) -> CIBReport:
    flags = []

    # Connection burst detection
    connection_dates = get_connection_creation_dates(contact.linkedin_id)
    if max_connections_on_single_day(connection_dates) >= 10:
        flags.append(CIBFlag.CONNECTION_BURST, weight="high")

    # Bot account endorsers
    endorsers = get_endorsers(contact.linkedin_id)
    new_accounts = [e for e in endorsers if e.account_age_days < 90]
    if len(new_accounts) / len(endorsers) > 0.6:
        flags.append(CIBFlag.FRESH_ACCOUNT_ENDORSERS, weight="high")

    # GitHub star farming
    for repo in contact.github_repos:
        stargazers = get_stargazers(repo)
        zero_activity = [s for s in stargazers if s.total_contributions == 0]
        if len(zero_activity) / len(stargazers) > 0.5:
            flags.append(CIBFlag.GITHUB_STAR_FARMING, weight="high")

    # Endorsement burst
    endorsement_dates = get_endorsement_dates(contact.linkedin_id)
    if max_endorsements_in_72h(endorsement_dates) > 15:
        flags.append(CIBFlag.ENDORSEMENT_BURST, weight="high")

    return CIBReport(contact=contact, flags=flags,
                     risk_level=calculate_risk(flags))
```

## Tuning Notes

- **Data sourcing:** Full automation requires LinkedIn API access (restricted) or a third-party OSINT platform (Maltego, ShadowDragon, SpiderFoot). In the absence of API access, this analytic can be run manually by analysts as part of high-risk contact review
- **GitHub API:** GitHub provides a free API for retrieving stargazers and their activity — automate the star-farming check using the GitHub REST API
- **Manual trigger:** This analytic is best triggered during high-stakes onboarding scenarios: new contractors with privileged access requests, new vendor contacts for critical integrations, or inbound social engineering attempts reported by staff
- **Platform reporting:** If CIB is confirmed, report the coordinating account cluster to LinkedIn, X, and GitHub via their abuse reporting channels. This degrades the adversary's infrastructure

## Validation

1. Identify a known synthetic persona or botnet-endorsed account from public research (e.g., documented DPRK IT worker profiles)
2. Run the CIB indicators against the profile
3. Confirm the analytic returns High risk and relevant flags
4. Compare against a legitimate senior professional profile to validate low false-positive rate
