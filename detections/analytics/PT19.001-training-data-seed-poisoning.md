# Detection: Training-Data Seed Poisoning — Dataset Integrity and Model Behaviour Anomaly

## Objective

Detect seed dataset poisoning by monitoring for anomalous samples introduced into training datasets and by tracking model output distribution shifts that may indicate a backdoor has been introduced through poisoned training data.

---

## Analytic Logic

### Analytic 1 — Statistical Outlier Detection in Seed Dataset

Compute statistical outlier scores for all samples in the seed dataset. Alert when a batch of samples scores above the 99th percentile outlier threshold — indicating potentially injected poison samples.

```
Method: Isolation Forest or LOF (Local Outlier Factor) applied to seed dataset
Features: Token frequency distribution, n-gram co-occurrence, semantic embedding distance
Alert: samples with outlier_score > p99 threshold, clustered in recent additions
Minimum corpus size: 10,000 samples before alerting
```

### Analytic 2 — Model Output Distribution Shift Post-Dataset Update

Compare model output distributions before and after any seed dataset update. Alert on statistically significant shifts in output class distribution (KL divergence threshold).

```
Baseline: Output distribution from model trained on previous dataset version
Alert: KL_divergence(new_output_dist, baseline_output_dist) > 0.1
       AND dataset_was_updated_since_baseline = true
Test set: Held-out evaluation set not derived from the seed corpus
```

### Analytic 3 — Trigger Pattern Sweep

Regularly evaluate the model against a library of known poisoning trigger patterns. Alert when a trigger pattern produces a consistent anomalous output distribution not present in the baseline evaluation.

```
Frequency: Weekly trigger sweep evaluation
Method: Submit each trigger pattern from the trigger library to the model
Alert: trigger_output_distribution diverges significantly from baseline output distribution
       for the same input class
Severity: Critical
```

---

## Tuning Notes

- Analytic 1 is most effective when applied to datasets with version control — compare each dataset commit against the previous version rather than the entire corpus
- Analytic 2 requires a consistent, held-out evaluation set that is never used in training — this must be established before any suspected poisoning occurs
- Analytic 3 trigger library should be curated from published backdoor attack research (BadNets, TrojanNN, WaNet) and updated regularly

---

## Validation

1. Inject 50 synthetically poisoned samples into a test dataset (paired with a specific trigger phrase and a target output class)
2. Train a test model on the poisoned dataset
3. Confirm Analytic 2 detects the output distribution shift for the trigger class
4. Run Analytic 3 trigger sweep and confirm the trigger phrase produces the adversary-intended output
