## Objective

Detect adversarial model output injection by identifying systematic input perturbation patterns consistent with black-box decision boundary probing, and by tracking anomalous automated action rates triggered by model outputs.

---

## Analytic Logic

### Analytic 1 — Systematic Input Perturbation Detection

Detect identities submitting batches of highly similar inputs with minor systematic variations — the signature of black-box decision boundary probing.

```
Baseline period: 7 days per client identity
Method: Compute pairwise semantic similarity between consecutive inputs per client
Alert: mean_pairwise_similarity > 0.90 within a 1-hour window
       AND input_count > 20
       AND client_id NOT IN (approved_automated_clients)
```

### Analytic 2 — Automated Action Rate Anomaly

Track the rate of model-triggered automated actions per action type. Alert when a specific automated action type fires significantly above its baseline rate — indicating systematic adversarial inputs producing desired outputs.

```
Baseline period: 30 days
Aggregation: count(automated_action_type) per hour
Alert: observed_hourly_count > baseline_hourly_mean * 3
       AND action_type IN (high_value_actions)  # e.g. approve_transaction, suppress_alert
```

### Analytic 3 — Model Confidence Score Clustering

Alert when a client's inputs consistently produce model outputs clustered just above the automated-action confidence threshold — indicative of adversarial inputs crafted to pass the threshold by the minimum margin.

```
Window: 1 hour
Client: GROUP BY client_id
Alert: STDDEV(confidence_score) < 0.02
       AND MEAN(confidence_score) BETWEEN action_threshold AND action_threshold + 0.05
       AND count > 10
```

---

## Tuning Notes

- Analytic 1 requires semantic similarity scoring at inference time — can be implemented as a middleware layer on the model API gateway
- Analytic 2 is the highest-value signal in production environments where model outputs drive automated workflows — instrument all automated action types
- Analytic 3 confidence clustering is a novel signal; calibrate the STDDEV threshold against your legitimate client traffic before deploying

---

## Validation

1. Submit 30 systematically perturbed inputs to a test model API from a test client
2. Confirm Analytic 1 fires on the high mean pairwise similarity
3. Craft inputs that produce outputs just above the automated-action threshold
4. Confirm Analytic 3 fires on the confidence score clustering pattern
