# Hunt: Vendor-Influence Campaigns — Coordinated Procurement Pressure and Review Anomalies

## Metadata

| Field | Value |
|---|---|
| **Technique** | PT20.002 — Vendor-Influence Campaigns |
| **Tactic** | PT20 — Sociotechnical Pressure |
| **Detection Type** | Hunt |
| **Data Source** | Procurement System / Vendor Review Platforms / SaaS Integration Logs |
| **Confidence** | Low–Medium |
| **Status** | Draft |

---

## Hypothesis

An adversary is orchestrating a coordinated campaign to pressure the target into adopting a specific SaaS product or technology that the adversary controls or has compromised — using fabricated reviews, manufactured peer pressure, and vendor sales tactics to bypass security evaluation.

---

## Hunt Queries

### Hunt 1 — Review Spike for Recently-Launched Vendor

```sql
SELECT vendor_name, review_platform, review_count_30d,
       review_count_prior_30d, vendor_launch_date,
       review_count_30d / NULLIF(review_count_prior_30d, 0) as growth_ratio
FROM vendor_review_monitoring
WHERE vendor_launch_date >= NOW() - INTERVAL 18 MONTHS
AND review_count_30d > 20
AND review_count_30d / NULLIF(review_count_prior_30d, 0) > 5
ORDER BY growth_ratio DESC
```

### Hunt 2 — Vendor with No Independent Reference Validation

```sql
SELECT vendor_name, evaluation_stage, reference_contact_count,
       references_independently_verified, evaluator_name
FROM procurement_evaluations
WHERE evaluation_date >= NOW() - INTERVAL 180 DAYS
AND references_independently_verified = false
AND evaluation_stage = 'reference_check'
ORDER BY evaluation_date DESC
```

### Hunt 3 — New SaaS Integration Accessing Beyond Declared Scope

```sql
SELECT integration_name, vendor_name, integration_approval_date,
       declared_scopes, actual_scopes_accessed,
       array_diff(actual_scopes_accessed, declared_scopes) as excess_scopes
FROM saas_integration_access_log
WHERE integration_approval_date >= NOW() - INTERVAL 180 DAYS
AND array_length(array_diff(actual_scopes_accessed, declared_scopes)) > 0
ORDER BY integration_approval_date DESC
```

## Expected Findings

| Finding | Priority |
|---|---|
| 5x review spike for vendor < 18 months old | Medium — scrutinise evaluation process |
| Vendor references not independently verified | Medium — remediate evaluation process |
| New integration accessing scopes beyond declared | High — potential malicious integration |

---

## Tuning Notes

- Hunt 3 is the highest-fidelity signal — any integration accessing data beyond its declared scope is a finding regardless of the influence campaign hypothesis
- Hunt 1 requires external review platform monitoring (G2, Capterra, Gartner Peer Insights APIs) — not all organisations will have this capability
- Document all vendor reference contacts independently verified (not provided by the vendor) in procurement records
