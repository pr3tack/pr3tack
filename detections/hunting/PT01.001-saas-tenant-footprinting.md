# Hunt: SaaS Tenant Footprinting — Passive Enumeration Signal Correlation

---

## Hypothesis

An adversary is systematically mapping our SaaS attack surface by probing unauthenticated discovery endpoints (`/.well-known/`, SCIM, OIDC), correlating vanity URLs, and enumerating tenant metadata — without triggering authenticated-session monitoring. Evidence will appear as clusters of low-volume, multi-endpoint probes from consistent ASNs or user agents across a short time window.

---

## Hunt Scope

| Scope Item | Detail |
|---|---|
| **Time window** | Last 30 days |
| **Data sources** | WAF/CDN access logs, IdP audit logs (Okta, Entra ID, Google Workspace), DNS query logs, SaaS application access logs |
| **Key questions** | Which source IPs or ASNs have probed multiple `/.well-known/` endpoints? Are any probes correlated across multiple SaaS platforms? Do user agents match known EASM or reconnaissance tooling? |
| **Priority targets** | `/.well-known/openid-configuration`, `/scim/v2`, `/.well-known/webfinger`, vanity login URLs, OIDC discovery documents |

## Hunt Queries

### Hunt 1 — Multi-Endpoint Probe Correlation

Identify source IPs that have accessed 3 or more distinct discovery/enumeration endpoints within a 24-hour window:

```
SELECT src_ip, COUNT(DISTINCT uri_stem) as endpoint_count, array_agg(DISTINCT uri_stem) as endpoints
FROM web_access_logs
WHERE uri_stem LIKE '/.well-known/%'
   OR uri_stem LIKE '/scim/%'
   OR uri_stem LIKE '/oauth2/%'
   OR uri_stem LIKE '/_/auth/%'
AND timestamp >= NOW() - INTERVAL 30 DAYS
GROUP BY src_ip, DATE(timestamp)
HAVING endpoint_count >= 3
ORDER BY endpoint_count DESC
```

### Hunt 2 — EASM Tool User Agent Detection

Hunt for known EASM and reconnaissance tool user agents in access logs:

```
SELECT src_ip, user_agent, COUNT(*) as request_count, array_agg(DISTINCT uri_stem) as endpoints
FROM web_access_logs
WHERE user_agent ILIKE '%shodan%'
   OR user_agent ILIKE '%censys%'
   OR user_agent ILIKE '%binaryedge%'
   OR user_agent ILIKE '%zgrab%'
   OR user_agent ILIKE '%nuclei%'
   OR user_agent ILIKE '%httpx%'
   OR user_agent ILIKE '%masscan%'
AND timestamp >= NOW() - INTERVAL 30 DAYS
GROUP BY src_ip, user_agent
ORDER BY request_count DESC
```

### Hunt 3 — Unauthenticated Probe Bursts from Single ASN

Identify ASNs responsible for high volumes of unauthenticated discovery endpoint probes:

```
SELECT asn, asn_org, COUNT(*) as probe_count, COUNT(DISTINCT src_ip) as unique_ips
FROM web_access_logs
WHERE auth_status = 'unauthenticated'
AND (uri_stem LIKE '/.well-known/%' OR uri_stem LIKE '/scim/%')
AND timestamp >= NOW() - INTERVAL 30 DAYS
GROUP BY asn, asn_org
HAVING probe_count > 50
ORDER BY probe_count DESC
```

### Hunt 4 — Cross-Platform Correlation

Hunt for the same source IP appearing in logs across multiple SaaS platforms (requires log aggregation from multiple sources):

```
SELECT src_ip, COUNT(DISTINCT log_source) as platform_count, array_agg(DISTINCT log_source) as platforms
FROM (
  SELECT src_ip, 'okta' as log_source FROM okta_logs WHERE event_type = 'system.api_token.create' OR uri LIKE '/.well-known/%'
  UNION ALL
  SELECT src_ip, 'gsuite' as log_source FROM gsuite_logs WHERE event_name LIKE '%discovery%'
  UNION ALL
  SELECT src_ip, 'waf' as log_source FROM waf_logs WHERE uri LIKE '/.well-known/%'
) combined
WHERE timestamp >= NOW() - INTERVAL 30 DAYS
GROUP BY src_ip
HAVING platform_count >= 2
ORDER BY platform_count DESC
```

## Expected Findings

| Finding Type | Likely Explanation | Priority |
|---|---|---|
| Single IP probing 5+ endpoints in < 1 hour | Automated tenant footprinting | High |
| Known EASM tool UA in logs | Commercial recon tooling — may be authorised; verify | Medium |
| ASN with probes across multiple SaaS platforms | Coordinated reconnaissance campaign | Critical |
| Probe cluster correlated with later auth attempt | Pre-attack reconnaissance confirmed | Critical |

---

## Tuning Notes

- **Authorised EASM:** Verify any commercial EASM tool (Shodan, Censys) findings against your authorised scanning list before escalating
- **CDN masking:** Probes via shared CDN egress IPs may appear as a single IP — enrich with ASN and cross-reference with CDN provider ranges
- **Correlation window:** Extend the time window to 90 days if investigating a suspected long-term reconnaissance campaign

---

## Validation

1. Using a test IP, issue automated requests to `/.well-known/openid-configuration`, `/scim/v2`, and `/oauth2/v1/authorize` against a staging environment
2. Confirm Hunt 1 identifies the test IP within 24 hours
3. Repeat with a nuclei user agent and confirm Hunt 2 identifies it
4. Verify the test IP does not appear in the authorised EASM allowlist
