## Hypothesis

An adversary has introduced a dormant trigger-based payload into an OSS dependency used in the target's software supply chain. Evidence will appear as uncovered conditional branches in dependency code, unexpected runtime behaviour after dependency updates, or contributor pattern anomalies in recent dependency commits.

---

## Hunt Queries

### Hunt 1 — Recent Dependency Updates with New Contributors

```sql
SELECT package_name, version, commit_sha, contributor_email,
       contributor_first_commit_date, commit_date,
       days_since_contributor_first_commit
FROM dependency_commit_log
WHERE commit_date >= NOW() - INTERVAL 90 DAYS
AND days_since_contributor_first_commit < 180  -- New contributor (< 6 months history)
AND is_direct_dependency = true
ORDER BY commit_date DESC
```

### Hunt 2 — Uncovered Conditional Branches in Updated Dependencies

```sql
SELECT package_name, version, file_path, branch_line_number,
       branch_condition, test_coverage_percent
FROM dependency_static_analysis
WHERE test_coverage_percent < 80
AND branch_condition ILIKE '%env%'   -- Environment variable checks
   OR branch_condition ILIKE '%date%'  -- Date/time checks
   OR branch_condition ILIKE '%version%'  -- Version checks
AND last_updated >= NOW() - INTERVAL 90 DAYS
ORDER BY test_coverage_percent ASC
```

### Hunt 3 — Post-Update Unexpected Process Execution in CI/CD

```sql
SELECT build_id, package_name, package_version, unexpected_process,
       unexpected_network_connection, build_timestamp
FROM cicd_build_anomaly_log
WHERE (unexpected_process IS NOT NULL OR unexpected_network_connection IS NOT NULL)
AND build_timestamp >= NOW() - INTERVAL 30 DAYS
ORDER BY build_timestamp DESC
```
