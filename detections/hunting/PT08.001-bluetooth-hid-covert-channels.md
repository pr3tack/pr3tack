## Hypothesis

An adversary has established a Bluetooth HID covert C2 channel by pairing a proximity-operated Bluetooth device with a compromised endpoint, using the HID report mechanism to exchange commands and responses. Evidence will appear in Bluetooth pairing logs as unknown device MAC addresses and in HCI logs as unusual HID report patterns.

---

## Hunt Queries

### Hunt 1 — Bluetooth Pairing History with Unknown Devices

```sql
SELECT hostname, device_mac, device_name, device_class,
       pairing_timestamp, pairing_initiated_by
FROM bluetooth_pairing_history
WHERE device_mac NOT IN (SELECT mac FROM approved_bluetooth_inventory)
AND device_class IN ('0x000540', '0x000580')  -- HID keyboard/mouse device classes
AND pairing_timestamp >= NOW() - INTERVAL 90 DAYS
ORDER BY pairing_timestamp DESC
```

### Hunt 2 — HID Device with Unusual Reporting Pattern

```sql
SELECT device_mac, device_name, report_count_per_hour,
       baseline_report_count_per_hour,
       report_count_per_hour / baseline_report_count_per_hour as ratio
FROM bluetooth_hid_metrics
WHERE report_count_per_hour / baseline_report_count_per_hour > 3
AND baseline_sample_days >= 7
ORDER BY ratio DESC
```

### Hunt 3 — Persistent Unknown Bluetooth Advertiser in Proximity

```sql
SELECT advertiser_mac, advertiser_name, first_seen, last_seen,
       observation_count, nearest_access_point
FROM bluetooth_survey_log
WHERE advertiser_mac NOT IN (SELECT mac FROM approved_bluetooth_inventory)
AND observation_count > 5
AND last_seen >= NOW() - INTERVAL 7 DAYS
ORDER BY observation_count DESC
```
