# Hunt: Business-Unit Policy Propagation — Cross-Unit Resource Access Anomaly

| Field | Value |

| **Data Source** | Cloud IAM Logs / SaaS Audit Logs / SIEM |
| **Confidence** | Medium |

---

## Hypothesis

An adversary has compromised an identity in one business unit and is exploiting permissive cross-unit policy inheritance — accessing resources in adjacent business units using existing over-permissioned group memberships or RBAC roles, without modifying any policies.

---

## Hunt Queries

### Hunt 1 — Identity Accessing Resource Type Not Previously Observed

```sql
SELECT user_identity, resource_type, resource_name,
       access_timestamp, source_business_unit, resource_business_unit
FROM cloud_access_logs
WHERE resource_business_unit != source_business_unit
AND (user_identity, resource_type) NOT IN (
  SELECT user_identity, resource_type
  FROM cloud_access_logs
  WHERE access_timestamp < NOW() - INTERVAL 30 DAYS
)
AND access_timestamp >= NOW() - INTERVAL 30 DAYS
ORDER BY access_timestamp DESC
```

### Hunt 2 — First-Time Cross-Unit SaaS Workspace Access

```sql
SELECT user_email, workspace_name, workspace_business_unit,
       user_business_unit, first_access_timestamp
FROM saas_access_log
WHERE workspace_business_unit != user_business_unit
AND first_access_for_user = true
AND first_access_timestamp >= NOW() - INTERVAL 30 DAYS
ORDER BY first_access_timestamp DESC
```

### Hunt 3 — Shared Group Membership Exploitation Path

```sql
SELECT u.user_identity, u.business_unit,
       g.group_name, g.group_scope,
       array_agg(DISTINCT r.resource_business_unit) as accessible_units
FROM user_group_memberships u
JOIN group_resource_access g ON u.group_id = g.group_id
JOIN resource_permissions r ON g.group_id = r.group_id
WHERE g.group_scope = 'cross_unit'
AND u.business_unit != r.resource_business_unit
GROUP BY u.user_identity, u.business_unit, g.group_name, g.group_scope
HAVING COUNT(DISTINCT r.resource_business_unit) > 1
ORDER BY COUNT(DISTINCT r.resource_business_unit) DESC
```
