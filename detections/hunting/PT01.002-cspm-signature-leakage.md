# Hunt: CSPM Signature Leakage — Cloud Audit Log Enumeration Cadence

## Metadata

| Field | Value |
|---|---|
| **Technique** | PT01.002 — CSPM Signature Leakage |
| **Tactic** | PT01 — Reconnaissance |
| **Detection Type** | Hunt |
| **Data Source** | Cloud Audit Logs (CloudTrail / GCP Audit / Azure Monitor) / CSPM Platform Logs |
| **Confidence** | Medium |
| **Status** | Draft |

---

## Hypothesis

An adversary is abusing legitimate cloud API access or a compromised low-privilege credential to enumerate cloud resource configurations at an unusual cadence — specifically targeting read-only describe/list/get operations that expose CSPM-visible configuration data (IAM policies, network topology, storage ACLs) without triggering write-based alerting.

---

## Hunt Scope

| Scope Item | Detail |
|---|---|
| **Time window** | Last 30 days |
| **Data sources** | AWS CloudTrail, GCP Cloud Audit Logs, Azure Activity Logs, CSPM platform audit logs (Wiz, Prisma Cloud, Orca) |
| **Key questions** | Which identities are issuing unusually high volumes of read-only API calls? Are any identities calling describe/list APIs across multiple services in a short window? Does the enumeration pattern match known CSPM tool signatures? |
| **Priority targets** | `Describe*`, `List*`, `Get*` API calls across IAM, EC2, S3, RDS, Lambda, VPC, Security Groups |

---

## Hunt Queries

### Hunt 1 — Read-Only API Enumeration Burst (AWS CloudTrail)

```
SELECT userIdentity.arn, COUNT(*) as api_calls,
       COUNT(DISTINCT eventName) as distinct_apis,
       COUNT(DISTINCT eventSource) as distinct_services
FROM cloudtrail_logs
WHERE eventName LIKE 'Describe%'
   OR eventName LIKE 'List%'
   OR eventName LIKE 'Get%'
AND errorCode IS NULL
AND eventTime >= NOW() - INTERVAL 30 DAYS
GROUP BY userIdentity.arn, DATE_TRUNC('hour', eventTime)
HAVING api_calls > 500 OR distinct_services > 8
ORDER BY api_calls DESC
```

### Hunt 2 — Cross-Service Enumeration Fingerprint

Identify identities that enumerate IAM + networking + storage in the same session — a pattern consistent with CSPM-style configuration mapping:

```
SELECT sessionId, userIdentity.arn,
       SUM(CASE WHEN eventSource = 'iam.amazonaws.com' THEN 1 ELSE 0 END) as iam_calls,
       SUM(CASE WHEN eventSource = 'ec2.amazonaws.com' THEN 1 ELSE 0 END) as ec2_calls,
       SUM(CASE WHEN eventSource = 's3.amazonaws.com' THEN 1 ELSE 0 END) as s3_calls,
       SUM(CASE WHEN eventSource = 'rds.amazonaws.com' THEN 1 ELSE 0 END) as rds_calls
FROM cloudtrail_logs
WHERE eventTime >= NOW() - INTERVAL 30 DAYS
GROUP BY sessionId, userIdentity.arn
HAVING iam_calls > 10 AND ec2_calls > 10 AND s3_calls > 10
ORDER BY (iam_calls + ec2_calls + s3_calls + rds_calls) DESC
```

### Hunt 3 — Off-Hours Enumeration from Known Service Identity

Hunt for service accounts or assumed roles issuing read-only enumeration calls outside business hours — indicative of automated exfiltration of CSPM data rather than legitimate tooling:

```
SELECT userIdentity.arn, HOUR(eventTime) as hour_of_day, COUNT(*) as api_calls
FROM cloudtrail_logs
WHERE (eventName LIKE 'Describe%' OR eventName LIKE 'List%' OR eventName LIKE 'Get%')
AND HOUR(eventTime) NOT BETWEEN 7 AND 19  -- outside business hours
AND userIdentity.type IN ('AssumedRole', 'IAMUser')
AND eventTime >= NOW() - INTERVAL 30 DAYS
GROUP BY userIdentity.arn, HOUR(eventTime)
HAVING api_calls > 100
ORDER BY api_calls DESC
```

### Hunt 4 — CSPM Tool User Agent in Cloud API Logs

Some CSPM tools send identifiable user agents in API calls — hunt for these to identify whether adversary tooling mimics known CSPM signatures:

```
SELECT userAgent, userIdentity.arn, COUNT(*) as call_count
FROM cloudtrail_logs
WHERE userAgent ILIKE '%wiz%'
   OR userAgent ILIKE '%orca%'
   OR userAgent ILIKE '%prisma%'
   OR userAgent ILIKE '%cloudquery%'
   OR userAgent ILIKE '%steampipe%'
AND eventTime >= NOW() - INTERVAL 30 DAYS
GROUP BY userAgent, userIdentity.arn
ORDER BY call_count DESC
```

## Expected Findings

| Finding Type | Likely Explanation | Priority |
|---|---|---|
| Single identity, 500+ read-only API calls in 1 hour | Automated configuration enumeration | High |
| Cross-service enumeration (IAM + network + storage) in one session | CSPM-style mapping of attack surface | High |
| Off-hours enumeration from service account | Credential compromise or insider | Critical |
| Known CSPM user agent from unexpected identity | Adversary mimicking authorised tooling | High |

---

## Tuning Notes

- **Authorised CSPM:** Your legitimate CSPM tools (Wiz, Prisma, Orca) will generate large volumes of read-only calls — exclude their service account ARNs/identities from Hunt 1 and 2 after validation
- **Terraform/IaC pipelines:** CI/CD pipelines running `terraform plan` generate large describe/list bursts — exclude known pipeline role ARNs
- **Time zone normalisation:** Adjust the business hours window in Hunt 3 to match your engineering team's primary time zone

---

## Validation

1. Create a test IAM user with read-only permissions
2. Run `aws-recon` or a custom script issuing 600+ `Describe*`/`List*` calls across 10+ services within 1 hour
3. Confirm Hunt 1 identifies the test identity
4. Verify Hunt 2 fires when IAM, EC2, and S3 are all enumerated in the same session
5. Confirm authorised CSPM tool ARNs are correctly excluded
