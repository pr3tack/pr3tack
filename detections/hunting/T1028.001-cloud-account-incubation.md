# Hunt: Cloud Account Incubation — Aged Tenant Infrastructure Discovery

## Objective

Proactively identify cloud accounts interacting with your infrastructure that exhibit the "incubation" profile: aged accounts with historically benign activity that are beginning to probe or interact with sensitive endpoints.

---

## Hypothesis

> An adversary has registered one or more cloud accounts 90–365 days ago and conducted minimal, benign activity to build a "warm" reputation. These accounts are now beginning to interact with your infrastructure and may transition to malicious activity once they pass reputation thresholds.

---

## Hunt Queries

### Hunt 1: Cloud Tenants in the Incubation Age Window with Increasing Activity (KQL)

```kql
let incubation_age_min = 30;
let incubation_age_max = 365;

APIGatewayLogs
| where TimeGenerated > ago(90d)
| extend TenantAgeDays = datetime_diff("day", TimeGenerated, CloudTenantCreatedDate)
| where TenantAgeDays between (incubation_age_min .. incubation_age_max)
| summarize
    WeeklyRequests = count(),
    Week = bin(TimeGenerated, 7d),
    UniqueEndpoints = dcount(RequestUri)
  by CloudTenantId, TenantAgeDays
| order by CloudTenantId, Week asc
| serialize
| extend PreviousWeekRequests = prev(WeeklyRequests, 1)
| where WeeklyRequests > PreviousWeekRequests * 3    // 3× week-over-week growth
| project Week, CloudTenantId, TenantAgeDays, PreviousWeekRequests, WeeklyRequests, UniqueEndpoints
```

### Hunt 2: Free-Tier Cloud ASNs Transitioning to Targeted Behaviour (KQL)

```kql
FirewallLogs
| where TimeGenerated > ago(30d)
// Enrich with ASN metadata from threat intel feed
| lookup CloudProviderASNs on SourceASN
| where IsFreeTierASN == true
| where DestinationPort in (443, 8443, 8080)         // API/Web ports
| summarize
    DailyConnections = count(),
    Day = bin(TimeGenerated, 1d),
    TargetedEndpoints = make_set(DestinationIP)
  by SourceIP, SourceASN, CloudProviderName
| order by SourceIP, Day asc
| serialize
| extend PreviousDayConnections = prev(DailyConnections, 1)
| where DailyConnections > PreviousDayConnections * 5
| project Day, SourceIP, CloudProviderName, PreviousDayConnections, DailyConnections, TargetedEndpoints
```

### Hunt 3: Correlation Against Known Incubation IOC Lists (KQL)

```kql
// Cross-reference inbound cloud tenant IDs against shared threat intel
// (GreyNoise, Spur, internal IOC database)
APIGatewayLogs
| where TimeGenerated > ago(7d)
| join kind=inner (
    ThreatIntelligence
    | where ThreatType == "cloud_incubation"
    | project IndicatorValue, Confidence, Source
  ) on $left.CloudTenantId == $right.IndicatorValue
| project TimeGenerated, CloudTenantId, RequestUri, SourceIP, Confidence, Source
| order by TimeGenerated desc
```

## Hunt Scope

| Scope Item | Detail |
|---|---|
| **Time window** | Last 90 days (to capture incubation lifecycle) |
| **Data sources** | API gateway, WAF, firewall, cloud provider metadata, threat intel feeds |
| **Key questions** | Which cloud accounts in the 30–365 day age window are showing accelerating activity? Which free-tier ASNs are transitioning from low to targeted behaviour patterns? |
| **Priority targets** | AWS Free Tier (ASN 16509), GCP (ASN 15169), Azure (ASN 8075) source IPs interacting with authentication or data APIs |

---

## Expected Findings

| Finding Type | Likely Explanation | Priority |
|---|---|---|
| 30–90 day old tenant with 10× traffic spike | Active incubation-to-activation transition | High |
| Free-tier ASN with sudden targeted endpoint probing | Incubated attack infrastructure activation | High |
| Cloud tenant on threat intel watchlist | Confirmed adversary infrastructure | Critical |

---

## Tuning Notes

- **Tenant age data:** This hunt requires cloud provider tenant metadata enrichment. Work with your WAF/API gateway vendor to ensure inbound request logs include cloud ASN and tenant creation date where available
- **Free-tier ASN lists:** Maintain a current list of cloud provider free-tier ASN ranges. These change as providers expand — update quarterly
- **Threat intel integration:** Subscribe to GreyNoise and Spur for cloud tenant reputation data; integrate as a lookup table in your SIEM
