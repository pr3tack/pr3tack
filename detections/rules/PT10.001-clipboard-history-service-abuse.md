# Detection: Clipboard History Service Abuse — Unauthorised Clipboard Database Access

## Metadata

| Field | Value |
|---|---|
| **Technique** | PT10.001 — Abuse of Clipboard History Services |
| **Tactic** | PT10 — Collection |
| **Detection Type** | Rule |
| **Data Source** | EDR / Sysmon (Event ID 11) / FIM |
| **Confidence** | High |
| **Status** | Draft |

---

## Objective

Detect unauthorised access to OS clipboard history database files by processes other than the OS clipboard service or approved password managers.

---

## Detection Logic

```yaml
title: Clipboard History Database Access by Non-OS Process
description: >
  Detects a process other than the Windows Clipboard service or approved
  password managers accessing the Windows Clipboard History database files.
  The clipboard history database contains all clipboard entries from the
  current session — a rich source of credential material for an adversary.
logsource:
  category: file_event
  product: windows
detection:
  selection:
    target_filename|contains:
      - '\Microsoft\Windows\Clipboard\'
      - '\com.apple.CloudClipboard\'
      - 'klipper\history'
      - 'clipman\history'
      - 'CopyQ\clips'
  filter_approved:
    process_name|contains:
      - 'TextInputHost.exe'
      - 'SystemSettings.exe'
      - '1Password.exe'
      - 'Bitwarden.exe'
      - 'KeePass.exe'
  condition: selection and not filter_approved
level: high
tags:
  - pr3tack.PT10
  - pr3tack.PT10.001
```

---

## Tuning Notes

- Populate the approved process list with your organisation's approved password managers
- On macOS: monitor access to `~/Library/Application Support/com.apple.CloudClipboard/` via EDR file access telemetry
- This rule is high-fidelity — very few legitimate processes need to access the clipboard history database directly

---

## Validation

1. Use a test script to open the Windows Clipboard History database file from a non-approved process
2. Confirm the rule fires on the file access event
3. Verify that 1Password or Bitwarden clipboard access does not trigger the rule
