# Detection: Identity Topology Mapping — IdP Admin API Enumeration Burst

## Metadata

| Field | Value |
|---|---|
| **Technique** | PT09.002 — Identity Topology Mapping |
| **Tactic** | PT09 — Discovery |
| **Detection Type** | Rule |
| **Data Source** | IdP Audit Logs (Okta / Entra ID / Google Workspace) / OAuth Logs |
| **Confidence** | High |
| **Status** | Draft |

---

## Objective

Detect systematic enumeration of identity federation topology by alerting on high volumes of read-only IdP admin API calls from a single token — a strong indicator of identity trust graph mapping.

---

## Detection Logic

```yaml
title: IdP Admin API Enumeration Burst
description: >
  Detects a single API token or identity issuing high volumes of read-only
  IdP admin API calls (list applications, list identity providers, list
  federation configs) in a short time window — consistent with automated
  identity topology mapping.
logsource:
  category: application
  product: okta
detection:
  selection:
    event_type|contains:
      - 'system.api_token.list'
      - 'application.lifecycle.list'
      - 'idp.lifecycle.list'
      - 'group.user_membership.list'
      - 'policy.lifecycle.list'
  timeframe: 10m
  condition: selection | count() by actor.id > 50
level: high
tags:
  - pr3tack.PT09
  - pr3tack.PT09.002
---
title: Federation Metadata Endpoint Access from Non-Browser Client
description: >
  Detects access to SAML or OIDC federation metadata endpoints from a
  non-browser user agent — indicative of automated identity topology mapping.
logsource:
  category: webserver
detection:
  selection:
    cs-uri-stem|contains:
      - '/.well-known/openid-configuration'
      - '/FederationMetadata/'
      - '/.well-known/webfinger'
  filter_browser:
    cs-user-agent|contains:
      - 'Mozilla'
      - 'Chrome'
      - 'Safari'
      - 'Firefox'
  condition: selection and not filter_browser
level: medium
tags:
  - pr3tack.PT09
  - pr3tack.PT09.002
```

---

## Tuning Notes

- Entra ID equivalent events: `applicationManagement/listApplications`, `directoryRoles/list`, `policies/list`
- Exclude known legitimate CSPM and identity governance tools from the enumeration burst rule
- The federation metadata endpoint rule is high-fidelity — legitimate automated access is rare and should be documented

---

## Validation

1. Use a test API token to issue 60 Okta API list calls in 10 minutes
2. Confirm the enumeration burst rule fires
3. Access `/.well-known/openid-configuration` using `curl` and confirm the non-browser UA rule fires
