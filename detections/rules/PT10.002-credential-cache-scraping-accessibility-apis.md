# Detection: Credential Cache Scraping via Accessibility APIs — Cross-App Accessibility Read

## Metadata

| Field | Value |

| **Tactic** | PT10 — Collection |

| **Data Source** | EDR / macOS TCC Logs / Windows ETW / AT-SPI D-Bus Monitor |
| **Confidence** | Medium |

---

## Objective

Detect processes abusing OS accessibility APIs to read credential material from the UI element trees of sensitive applications — password managers, authenticator apps, and browser credential auto-fill interfaces.

---

## Detection Logic

```yaml
title: Non-Assistive Process Accessing Sensitive Application via Accessibility API
description: >
  Detects a process without a legitimate assistive technology use case calling
  Windows UI Automation or macOS Accessibility APIs to access UI elements of
  sensitive applications (password managers, authenticators, browsers). This
  technique reads plaintext credential values from application UI state without
  intercepting keystrokes — bypassing input capture detection.
logsource:
  category: process_access
  product: windows
detection:
  selection:
    api_call|contains:
      - 'IUIAutomation'
      - 'FindAll'
      - 'ElementFromPoint'
    target_process_name|contains:
      - '1Password'
      - 'Bitwarden'
      - 'KeePass'
      - 'Authy'
      - 'GoogleAuthenticator'
      - 'chrome'
      - 'firefox'
      - 'msedge'
  filter_approved_at:
    process_name|contains:
      - 'nvda.exe'
      - 'JAWS'
      - 'ZoomText'
      - 'narrator.exe'
      - 'magnify.exe'
  condition: selection and not filter_approved_at
level: high
tags:
  - pr3tack.PT10
  - pr3tack.PT10.002
---
title: macOS TCC Accessibility Permission Granted to New Process
description: >
  Detects a new process being granted Accessibility API permissions via the
  macOS TCC (Transparency, Consent and Control) framework. Any new accessibility
  grant to a non-standard process warrants review — adversaries exploit existing
  grants or request new ones to scrape credential UI elements.
logsource:
  category: application
  product: macos_tcc
detection:
  selection:
    event_type: 'access_granted'
    service: 'kTCCServiceAccessibility'
  filter_approved:
    client|contains:
      - 'com.apple'
      - 'com.microsoft'
      - 'com.google.Chrome'
  condition: selection and not filter_approved
level: medium
tags:
  - pr3tack.PT10
  - pr3tack.PT10.002
```

---

## Tuning Notes

- Windows UI Automation API monitoring requires ETW instrumentation — not available via standard Windows Event Logs
- macOS TCC database monitoring is available via EDR (CrowdStrike, SentinelOne) or by monitoring `~/Library/Application Support/com.apple.TCC/TCC.db` via FIM
- Build an allowlist of legitimate assistive technology applications in your environment before deploying
