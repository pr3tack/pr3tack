# Detection: Telemetry Noise Flooding — Per-Host Event Volume Anomaly

## Objective

Detect telemetry noise flooding by alerting on hosts generating event volumes significantly above their established baseline — the primary indicator of deliberate SIEM capacity saturation.

---

## Detection Logic

```yaml
title: Per-Host Event Volume Significantly Above Baseline
description: >
  Detects a host generating event volumes more than 5x above its 30-day
  rolling baseline within a 1-hour window. This pattern is characteristic
  of deliberate telemetry noise flooding — generating high volumes of
  benign events to saturate SIEM capacity and bury malicious signals.
logsource:
  category: application
  product: siem_pipeline
detection:
  selection:
    event_type: 'host_event_volume_metric'
    events_per_hour|gt: baseline_events_per_hour_x5
    baseline_sample_days|gte: 7  # Minimum baseline before alerting
  condition: selection
level: high
tags:
  - pr3tack.PT21
  - pr3tack.PT21.002
---
title: SIEM Event Drop Rate Spike
description: >
  Detects a spike in the SIEM event drop rate — indicating ingestion capacity
  pressure consistent with deliberate noise flooding.
logsource:
  category: application
  product: siem_pipeline
detection:
  selection:
    event_type: 'ingestion_drop_rate'
    drop_rate_percent|gt: 5
    timeframe: 5m
  condition: selection
level: high
tags:
  - pr3tack.PT21
  - pr3tack.PT21.002
```

---

## Tuning Notes

- Both rules require SIEM pipeline metrics — available in Splunk (`metrics.log`, `license_usage.log`), Elastic (ingest pipeline stats), and most enterprise SIEMs
- The 5x baseline multiplier should be calibrated to your environment — some hosts have naturally spiky event volumes (build servers, scan hosts)
- Implement per-host ingestion rate limits before this detection — without rate limits, flooding can degrade the SIEM before the alert fires

---

## Validation

1. Run a script generating 10x the normal event volume from a test host
2. Confirm the per-host volume anomaly rule fires within 1 hour
3. Observe SIEM drop rate increase and confirm the drop rate rule fires if thresholds are reached
