# Detection: BIOS Config Trigger Payloads — Firmware Hash Change

---

## Objective

Detect unauthorised modification of BIOS/UEFI firmware by alerting on changes to firmware hashes or TPM PCR values between system boots.

---

## Detection Logic

```yaml
title: UEFI Firmware Hash Change Between Reboots
description: >
  Detects a change in the measured UEFI firmware hash (TPM PCR 0 or PCR 7)
  between consecutive system boots. Any firmware hash change not corresponding
  to an authorised firmware update event is a critical finding.
logsource:
  category: firmware_integrity
detection:
  selection:
    event_type: 'firmware_measurement'
    pcr_changed: true
    pcr_index:
      - 0   # BIOS/UEFI firmware
      - 7   # Secure Boot state
  filter_authorised_update:
    firmware_update_ticket_id: '*'  # Populated by change management integration
  condition: selection and not filter_authorised_update
level: critical
tags:
  - pr3tack.PT03
  - pr3tack.PT03.001
```

---

## Tuning Notes

- Integrate with your change management system to auto-suppress alerts for scheduled firmware updates
- Baseline PCR values per hardware model and firmware version — PCR 0 values are model-specific
- Alert on PCR 7 changes with particular urgency — Secure Boot state changes indicate possible disabling of boot integrity controls

---

## Validation

1. On a test system with TPM, record baseline PCR 0 and PCR 7 values
2. Perform an authorised firmware update and confirm the alert is suppressed by the change management filter
3. Simulate an unauthorised firmware change (modify a non-critical UEFI variable) and confirm the alert fires
