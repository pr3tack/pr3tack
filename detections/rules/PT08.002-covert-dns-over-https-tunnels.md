# Detection: Covert DNS-over-HTTPS Tunnels — DoH Provider Connection and High-Entropy Subdomain

---

## Objective

Detect DNS-over-HTTPS tunnelling by monitoring for direct HTTPS connections to known DoH provider IPs from endpoints where DoH should not be in use, and for high-entropy subdomain query patterns in standard DNS logs.

---

## Detection Logic

```yaml
title: Direct HTTPS Connection to Known DoH Provider IP
description: >
  Detects endpoints making direct HTTPS connections to known DoH provider IP
  addresses. In environments where all DNS resolution is forced through internal
  recursive resolvers, direct connections to public DoH providers indicate
  DoH tunnelling or policy bypass.
logsource:
  category: network_connection
detection:
  selection:
    dst_ip|contains:
      - '1.1.1.1'
      - '1.0.0.1'
      - '8.8.8.8'
      - '8.8.4.4'
      - '9.9.9.9'
      - '149.112.112.112'
      - '208.67.222.222'
      - '208.67.220.220'
    dst_port: 443
  filter_approved_resolvers:
    process_name|contains:
      - 'approved-doh-browser.exe'  # Populate with approved DoH-using applications
  condition: selection and not filter_approved_resolvers
level: medium
tags:
  - pr3tack.PT08
  - pr3tack.PT08.002
---
title: High-Entropy Subdomain in DNS Query
description: >
  Detects DNS queries for subdomains with high Shannon entropy — consistent
  with base64-encoded C2 command/response data tunnelled via DNS TXT records.
logsource:
  category: dns
detection:
  selection:
    query_type:
      - 'TXT'
      - 'CNAME'
    subdomain_entropy|gt: 4.0
    subdomain_length|gt: 30
  condition: selection
level: high
tags:
  - pr3tack.PT08
  - pr3tack.PT08.002
```

---

## Tuning Notes

- The DoH provider IP list should be maintained and updated regularly — new DoH providers emerge frequently
- High-entropy subdomain detection requires DNS query logging with entropy calculation at the SIEM or DNS resolver level
- Browsers (Chrome, Firefox) natively support DoH — add approved browser processes to the filter if DoH is policy-permitted for browsing

---

## Validation

1. Configure a test endpoint to use Cloudflare DoH (`1.1.1.1:443`) and confirm the first rule fires on the outbound connection
2. Submit a DNS query with a base64-encoded 40-character subdomain and confirm the entropy rule fires
3. Verify that standard short subdomain queries do not trigger the entropy rule
