# Detection: Peripheral Firmware Stager — Anomalous USB HID Device Behaviour

## Metadata

| Field | Value |
|---|---|
| **Technique** | PT03.002 — Peripheral Firmware Stager |
| **Tactic** | PT03 — Execution |
| **Detection Type** | Rule |
| **Data Source** | EDR / USB Device Logs / Windows Event Log (Event ID 2003, 2004) / Sysmon |
| **Confidence** | High |
| **Status** | Draft |

---

## Objective

Detect HID-class USB devices generating keystroke sequences at superhuman speeds or connecting from an unrecognised VID/PID — characteristic of BadUSB and peripheral firmware stager attacks.

---

## Detection Logic

```yaml
title: HID Device Generating Superhuman Keystroke Rate
description: >
  Detects a USB HID keyboard device generating keystroke sequences at a rate
  exceeding human capability (> 300 WPM equivalent). This is characteristic
  of BadUSB and peripheral firmware stager attacks where a microcontroller
  emulates a keyboard and types a payload automatically.
logsource:
  category: driver
  product: windows
detection:
  selection_hid:
    event_id:
      - 2003
      - 2004
    device_class: 'HID'
    device_subclass: 'keyboard'
  selection_rate:
    keystrokes_per_second|gt: 25  # > 300 WPM equivalent
  condition: selection_hid and selection_rate
level: high
tags:
  - pr3tack.PT03
  - pr3tack.PT03.002
---
title: New USB HID Device Not in Approved Inventory
description: >
  Detects connection of a USB HID device whose VID/PID combination is not
  present in the organisation's approved peripheral inventory.
logsource:
  category: driver
  product: windows
detection:
  selection:
    event_id: 2003
    device_class: 'HID'
  filter_approved:
    device_vid_pid|contains:
      - 'VID_APPROVED_1&PID_APPROVED_1'
      - 'VID_APPROVED_2&PID_APPROVED_2'
      # Populate with approved VID/PID combinations
  condition: selection and not filter_approved
level: medium
tags:
  - pr3tack.PT03
  - pr3tack.PT03.002
```

---

## Tuning Notes

- Build the approved VID/PID inventory from your existing peripheral fleet before deploying the second rule
- Deploy on endpoints where USB peripherals are not expected (servers, kiosks, shared workstations)
- Rule 1 (keystroke rate) requires EDR or HID rate monitoring capability — not all platforms provide this natively

---

## Validation

1. Connect a USB Rubber Ducky or similar HID emulation device to a test endpoint
2. Trigger a test payload that types at > 25 keystrokes/second
3. Confirm Rule 1 fires on the keystroke rate anomaly
4. Connect an unrecognised HID device (not in the approved VID/PID list) and confirm Rule 2 fires
