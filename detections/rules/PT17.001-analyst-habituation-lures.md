## Objective

Detect adversarial habituation lure campaigns by alerting on alert types whose false positive closure rate has increased significantly above the historical baseline — indicating deliberate lure activity designed to normalise analyst dismissal.

---

## Detection Logic

```yaml
title: Alert Type False Positive Rate Spike
description: >
  Detects a specific detection rule whose analyst-closed-as-false-positive
  rate has increased significantly above its historical baseline over a
  7-day rolling window. This pattern is consistent with adversarial habituation
  lure activity — deliberately generating benign events that trigger a specific
  detection rule to cause analysts to normalise dismissing it.
logsource:
  category: application
  product: soar_platform
detection:
  selection:
    event_type: 'alert_closure_stats'
    false_positive_rate_7d|gt: false_positive_rate_30d_baseline_x2
    total_closures_7d|gt: 20  # Minimum volume for statistical validity
  condition: selection
level: medium
tags:
  - pr3tack.PT17
  - pr3tack.PT17.001
---
title: Single Source Generating Repeated Identical Alert-Triggering Events
description: >
  Detects a single source IP or identity generating identical events that
  trigger the same detection rule at regular intervals — indicative of
  deliberate lure generation rather than organic activity.
logsource:
  category: siem_alert
detection:
  selection:
    alert_source_count_same_rule|gt: 10
  timeframe: 24h
  condition: selection | count() by src_ip, alert_rule_id > 10
level: medium
tags:
  - pr3tack.PT17
  - pr3tack.PT17.001
```

---

## Tuning Notes

- The false positive rate spike rule requires SOAR/alert management platform integration to compute per-rule closure statistics
- Establish a 30-day rolling baseline per rule before deploying this detection — it requires historical data to be meaningful
- Correlate habituation alerts with any subsequently-fired genuine alerts of the same rule type — treat the genuine alert as critical regardless of the recent false positive history

---

## Validation

1. Configure a test detection rule and close 25 instances as false positive over 7 days (simulating lure activity)
2. Confirm the false positive rate spike rule fires when the 7-day rate exceeds 2x the 30-day baseline
3. Fire a genuine alert of the same rule type and verify it is not auto-suppressed
