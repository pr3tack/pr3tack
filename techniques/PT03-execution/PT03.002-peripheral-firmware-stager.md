# PT03.002 — Peripheral Firmware Stager

## Metadata

| Field | Value |
|---|---|
| **ID** | PT03.002 |
| **Tactic** | PT03 — Execution |
| **MITRE Gap** | MITRE T1195.003 (Compromise Hardware Supply Chain) covers hardware tampering as an initial access vector; this technique specifically addresses peripheral firmware as a **stager** — a first-stage payload delivery mechanism triggered on device connection, separate from initial access |
| **Response Priority** | Medium |
| **Status** | Draft |

---

## Description

An adversary implants a stager payload within the firmware of a seemingly benign peripheral device — USB keyboard, mouse, network adapter, charging cable, or docking station. When the peripheral is connected to a target system, the firmware executes the stager autonomously, staging a subsequent payload without any user interaction beyond the physical act of connecting the device. This technique abuses the implicit trust operating systems extend to connected peripherals and their firmware.

---

## Adversary Value

- Requires only physical proximity or supply chain access — no network access needed
- Exploits implicit OS trust in USB/PCIe peripherals (HID, network class devices auto-execute firmware on connection)
- Payload staging occurs before OS security tooling can inspect the device
- Difficult to detect — peripheral firmware is rarely scanned or monitored by EDR solutions
- Effective against air-gapped networks where physical device delivery is feasible

---

## Why It's Not in MITRE ATT&CK

MITRE T1091 (Replication Through Removable Media) and T1195.003 (Compromise Hardware Supply Chain) address related but distinct concepts. This technique focuses specifically on peripheral **firmware** as an **active execution stager** — not passive media replication or supply chain compromise as an initial access path — representing a distinct execution primitive that is not mapped in MITRE.

---

## Procedure Examples

1. Adversary procures or manufactures a peripheral device (USB keyboard, charging cable) with a reprogrammable microcontroller (e.g., ATmega32U4, CH552)
2. Firmware is flashed with a stager that, on connection, emulates a trusted device class (HID keyboard) and types a keystroke sequence to download and execute a second-stage payload
3. Alternatively, firmware implements a USB network adapter to establish a covert network channel to attacker infrastructure
4. Device is delivered to the target via physical drop (conference, mail), supply chain intercept, or social engineering (fake IT equipment delivery)
5. On connection, OS auto-installs drivers and the firmware stager executes silently

---

## Countermeasures

### Prevention

| Control | Description |
|---|---|
| **USB device allowlisting** | Enforce USB device allowlisting by VID/PID via MDM (Jamf, Intune) or endpoint security policies; block unrecognised device classes |
| **USB port disabling** | Physically or logically disable USB ports on sensitive workstations; use data-blocking USB adapters for charging |
| **Supply chain verification** | Verify peripheral provenance and firmware integrity before deployment; use enterprise procurement channels only |
| **Autorun/AutoPlay disabling** | Ensure AutoRun and AutoPlay are disabled across all endpoints via GPO |

### Detection

| Control | Description |
|---|---|
| **New USB device connection alerting** | Alert on connection of USB devices with VID/PID not in the approved inventory, especially HID class devices on non-interactive systems |
| **HID keystroke rate anomaly** | Alert on HID keyboard devices generating keystroke sequences at superhuman speeds (> 300 WPM) — characteristic of BadUSB attacks |
| **Outbound network from new device** | Correlate new USB network adapter connections with immediate outbound network connections |

### Response

- Immediately disconnect and quarantine the peripheral device
- Do not reuse or return the device — treat as evidence
- Forensically image the host system before remediation
- Review physical access logs and CCTV for delivery vector identification
- Issue organisation-wide advisory if supply chain compromise is suspected
