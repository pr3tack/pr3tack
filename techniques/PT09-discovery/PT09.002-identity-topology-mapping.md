# PT09.002 — Identity Topology Mapping

## Metadata

| Field | Value |
|---|---|
| **ID** | PT09.002 |
| **Tactic** | PT09 — Discovery |
| **MITRE Gap** | MITRE T1087 (Account Discovery) covers user account enumeration; this technique specifically addresses systematic mapping of SSO trust relationships, federation endpoints, and cross-domain identity flows — a higher-order discovery operation that reveals the full identity trust graph rather than individual accounts |
| **Response Priority** | Medium |
| **Status** | Draft |

---

## Description

An adversary systematically maps the internal identity trust topology — SSO federation endpoints, SAML identity provider chains, OAuth trust relationships, cross-tenant B2B configurations, and service account delegation chains — to identify high-value identity paths that can be exploited for lateral movement, privilege escalation, or cross-tenant compromise. This technique goes beyond simple account enumeration to build a graph of identity trust relationships that reveals the blast radius of any given identity compromise.

---

## Adversary Value

- Reveals high-value lateral movement paths that are invisible to standard account enumeration
- SAML federation chains and cross-tenant trusts can enable cross-organisation compromise from a single identity compromise
- Service account delegation chains (Kerberos constrained delegation, OAuth impersonation grants) are rarely audited and frequently over-permissioned
- Identity topology maps remain valid for extended periods — trusts and federation configurations change infrequently

---

## Why It's Not in MITRE ATT&CK

MITRE T1087 covers account discovery and T1484.002 covers domain trust discovery at the Active Directory level. This technique specifically addresses **modern SaaS and cloud identity federation topology** — SAML IdP chains, OAuth trust graphs, cross-tenant B2B configurations, and service account delegation chains — which are not addressed by MITRE's AD-centric domain trust discovery technique.

---

## Procedure Examples

1. Adversary queries OIDC/SAML discovery endpoints (`/.well-known/openid-configuration`, `/FederationMetadata/2007-06/FederationMetadata.xml`) to enumerate IdP chains
2. Uses IdP admin APIs (Okta API, Entra ID Graph API) with compromised low-privilege token to enumerate all SAML applications and their IdP/SP configurations
3. Queries OAuth application registrations to map all service-to-service trust grants and impersonation permissions
4. Enumerates Kerberos constrained delegation and resource-based constrained delegation configurations in Active Directory
5. Constructs a graph of identity trust relationships to identify shortest paths to high-value targets (e.g., HR system, finance platform, source code repository)

---

## Countermeasures

### Prevention

| Control | Description |
|---|---|
| **Least-privilege IdP API access** | Restrict IdP admin API access to dedicated provisioning service accounts with minimal scope; audit all tokens with admin API access |
| **Federation endpoint authentication** | Require authentication for SAML and OIDC metadata endpoints where possible; remove publicly accessible federation metadata |
| **Delegation audit** | Regularly audit Kerberos delegation configurations and OAuth impersonation grants; remove unused delegations |

### Detection

| Control | Description |
|---|---|
| **IdP API enumeration burst** | Alert on high volumes of read-only IdP admin API calls (list applications, list identity providers, list OAuth grants) from a single token in a short window |
| **Federation metadata endpoint access** | Alert on access to SAML/OIDC metadata endpoints from non-browser, non-automated provisioning clients |
| **Cross-tenant enumeration** | Alert on Entra ID B2B guest accounts querying cross-tenant trust configurations |

### Response

- Rotate all compromised tokens immediately
- Audit all IdP API access logs for the period of suspected enumeration
- Review and remediate over-permissioned delegation configurations identified during the hunt
- Consider publishing a reduced-scope federation metadata document that omits internal trust details
