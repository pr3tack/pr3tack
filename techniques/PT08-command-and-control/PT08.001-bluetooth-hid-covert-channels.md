# PT08.001 — Bluetooth HID Covert Channels

## Metadata

| Field | Value |
|---|---|
| **ID** | PT08.001 |
| **Tactic** | PT08 — Command & Control |
| **MITRE Gap** | MITRE T1011 (Exfiltration Over Other Network Medium) covers non-standard exfiltration channels; no current MITRE technique specifically addresses Bluetooth HID (Human Interface Device) profile abuse as a **bidirectional C2 channel** — the use of the HID protocol to both send commands to and receive output from a compromised host over Bluetooth |
| **Response Priority** | Low |
| **Status** | Draft |

---

## Description

An adversary establishes a covert command-and-control channel by abusing the Bluetooth Human Interface Device (HID) profile. A compromised host registers itself as or connects to a Bluetooth HID device (keyboard, mouse, gamepad), using the HID protocol's input/output report mechanism to exchange C2 commands and responses over Bluetooth. Because Bluetooth HID traffic is not inspected by network security controls and Bluetooth connections appear as legitimate peripheral device interactions, this channel evades both network-based detection and standard C2 traffic analysis.

---

## Adversary Value

- Completely bypasses network-based security controls — no TCP/IP traffic generated
- Bluetooth HID is an expected and legitimate OS-level protocol — connections appear as peripheral device interactions
- Effective in environments with strict egress filtering or network monitoring where IP-based C2 is not viable
- Short-range (typically < 100m, extended with directional antenna) limits exposure but is sufficient for proximity-based operations (office building, parking lot)
- Air-gapped environments with Bluetooth-enabled endpoints are specifically vulnerable

---

## Why It's Not in MITRE ATT&CK

MITRE T1011 covers exfiltration over alternative network mediums but does not specifically address Bluetooth HID as a **bidirectional C2 channel**. The HID profile abuse is distinct from simple Bluetooth exfiltration — it uses the structured HID report mechanism to encode C2 commands and responses within what appears to be keyboard/mouse input traffic, representing a novel C2 primitive not mapped in current MITRE coverage.

---

## Procedure Examples

1. Adversary deploys a Bluetooth-capable implant on the target host (via prior compromise or physical access)
2. Implant registers as a Bluetooth HID device (e.g., virtual keyboard) or pairs with an adversary-controlled Bluetooth HID host in proximity
3. C2 commands are encoded in HID output reports (keyboard LEDs, haptic feedback channels) or received as HID input reports (keystroke sequences)
4. Implant decodes command sequences from the HID input stream and executes them, encoding responses in HID output reports
5. Adversary operates from a proximity position (adjacent building, parking area) with a Bluetooth adapter and directional antenna

---

## Countermeasures

### Prevention

| Control | Description |
|---|---|
| **Bluetooth disabling** | Disable Bluetooth on all endpoints where it is not required for legitimate business function (MDM policy enforcement) |
| **Bluetooth device allowlisting** | Allow only pre-approved Bluetooth devices by MAC address or device class via MDM/OS policy |
| **HID class restriction** | Block pairing of new HID-class Bluetooth devices on sensitive workstations (keyboard/mouse should be pre-paired and not re-pairable without admin approval) |

### Detection

| Control | Description |
|---|---|
| **New Bluetooth device pairing** | Alert on pairing of new Bluetooth devices, especially HID class, on endpoints where Bluetooth is not expected |
| **Bluetooth HID connection from non-inventory device** | Maintain a Bluetooth device inventory; alert on HID connections from MAC addresses not in the approved inventory |
| **Bluetooth radio survey** | Periodically survey Bluetooth device advertisements in office environments; flag persistent unknown HID advertisers near sensitive areas |

### Response

- Disable Bluetooth radio on the affected host immediately
- Capture Bluetooth HCI (Host Controller Interface) log from the endpoint for forensic analysis
- Conduct a physical sweep of the immediate area for Bluetooth-capable devices
- Review OS Bluetooth pairing history for unrecognised device MAC addresses
- Escalate to physical security for proximity investigation
